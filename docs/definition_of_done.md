# Definition of Done

A story is done when:

- Acceptance criteria are met
- Code changes are integrated into the target branch
- Relevant edge cases are handled
- Automated tests pass
- UI behaviour is verified when the story affects the interface
- Logging, audit, or evaluation behaviour is validated when the story affects those areas
- Documentation is updated when the story changes product behaviour, assumptions, metrics, or setup instructions
- Required project artifacts are added or updated
- Temporary, local, generated, or sensitive files are excluded from version control
