# POL-007-DOCUMENTATION-STANDARDS — DOCUMENTATION AND ADVISOR NOTE STANDARDS

Document Owner: Compliance Operations
Version: 3.1
Last Updated: April 2025
Internal Distribution: Advisory, Compliance, Branch Managers, Audit

---

# 1. Purpose

This policy establishes minimum standards for advisor documentation supporting investment recommendations.

Adequate documentation enables effective supervisory review and supports audit readiness.

Documentation deficiencies may create supervisory concerns regardless of underlying trade suitability.

---

# 2. General Documentation Principles

Advisor notes should accurately reflect:

* client discussions,
* suitability considerations,
* material risks,
* and recommendation rationale.

Documentation should be completed contemporaneously whenever reasonably possible.

Supervisors should be able to understand the basis for a recommendation without relying on assumptions.

---

# 3. Minimum Documentation Expectations

Advisor notes should generally include:

* relevant client objectives,
* risk tolerance considerations,
* investment horizon,
* material risks discussed,
* rationale for the recommendation,
* and any significant suitability considerations.

Additional documentation may be required for higher-risk recommendations.

---

# 4. Inadequate Documentation Examples

The following examples may be considered insufficient without additional detail:

* "client approved trade,"
* "client requested purchase,"
* "growth opportunity,"
* "long-term investment,"
* "client understands risk,"
* or "portfolio rebalance."

Documentation should explain why the recommendation is appropriate for the specific client.

---

# 5. Enhanced Documentation Situations

Enhanced documentation may be required where:

* high-risk products are recommended,
* concentration risk exists,
* elderly clients are involved,
* significant KYC changes occur,
* speculative investments are purchased,
* or suitability concerns require additional explanation.

Advisor notes should specifically address the factors that increase supervisory risk.

---

# 6. Missing Documentation

Missing rationale may itself constitute a supervisory concern.

Where documentation is incomplete, supervisors may require:

* advisor clarification,
* supplemental notes,
* enhanced review,
* or escalation.

The absence of documentation should not be interpreted as evidence that suitability discussions occurred.

---

# 7. Escalation Triggers

The following situations generally require additional review:

* absent advisor notes,
* generic rationale,
* contradictory documentation,
* missing suitability discussion,
* unresolved KYC inconsistencies,
* or repeated documentation deficiencies.

Supervisors may require remediation where documentation standards are not met.

---

# 8. Supervisory Review Notes

Documentation supports both compliance oversight and client protection.

Advisor notes should clearly demonstrate the reasoning process supporting the recommendation.

Well-documented recommendations improve supervisory efficiency and strengthen audit defensibility.
